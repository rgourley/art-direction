Create a Highlight

A Highlight is a range on a document, plus everything you need to find it again after the document changes. The range is stored as a triple: (a) the exact substring you selected, (b) the DOM path to its ancestor, (c) enough surrounding context to reanchor when either shifts. Store all three, resolve in that order.

The version of your document that existed at highlight time is gone. The API's job is to find the closest equivalent range in the version that exists now. Design memo, 2026-01
POST /v2/highlights
curl https://api.marker.dev/v2/highlights \
  -H "Authorization: Bearer $MARKER_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "page_url": "https://marker.dev/blog/reanchoring",
    "exact": "the version of your document",
    "prefix": "That is,",
    "suffix": "that existed at highlight time",
    "container": "main > article > p:nth-of-type(3)",
    "color": "#C7FF37"
  }'
# → 201 Created, returns { id, resolved_range, confidence: 1.0 }

The response includes a confidence score. A value of 1.0 means the exact string was found in the container element on the current version of the page. Anything less means the API used the prefix/suffix triple to reanchor to the nearest match. Store the score with the highlight; render UI treatment based on it (a subtle warning when confidence < 0.85, a full "this may have moved" flag when < 0.5).

Request parameters
NameTypeRequiredDescription
page_url string yes Absolute URL of the document being annotated. Fragment identifiers stripped before storage.
exact string yes The selected substring, verbatim. Whitespace preserved. Max 500 chars.
prefix string opt. Up to 32 chars immediately preceding the selection. Improves reanchor accuracy when the container changes.
suffix string opt. Up to 32 chars immediately following the selection. Together with prefix, forms the disambiguator.
container selector opt. CSS selector for the closest identifiable ancestor. Used first when resolving; falls back to prefix/suffix on miss.
color hex opt. Any 6-digit hex, alpha ignored. Defaults to the workspace's brand accent.

Learn about resolution failures in § 4.4 Reanchoring and how the webhook fires when a highlight's confidence drops below your threshold in § 6 Webhooks. If you're building a reader that needs sub-100ms highlight loading, see the bulk fetch endpoint.
